# 🎨 MySave Design System

## 1. 디자인 전략

MySave는 웹 서핑의 맥락을 방해하지 않는 자연스러운 수집 과정과, 수많은 정보 속에서도 원하는 것을 망설임 없이 발견하는 탐색 경험을 설계합니다. 기존 북마크의 정적인 리스트 방식을 탈피하여, 사용자의 사고 체계에 맞춘 유연한 정보 관리 환경을 제공합니다.

### 1-1) UX 목표

사용자가 정보를 수집하고 관리하는 과정에서 겪는 실제적인 불편함을 디자인적 사고로 해결하는 데 집중했습니다.

| 사용자 문제  | 해결 방안 | 핵심 가치  |
| --- | --- | --- |
| 복잡한 북마크 과정  | Chrome Extension | 사용 편리성 |
| 저장할 때마다 흐름이 끊김 | 단 한번의 클릭으로 자동 저장  | 방해 없는 수집 환경  |
| 텍스트 위주의 지루한 리스트  | 카드형 UI | 시각적 명확성 |
| 원하는 정보를 한눈에 찾기 어려움 | 이미지와 태그 중심의 시각적 배치 | 탐색 인지 및 부담 시간 감소 |
| 잊혀지는 정보들  | 리마인더 & 태그 시스템  | 지속 가능성 |
| 저장만 해두고 다시 보지 않음  | 타임라인과 알림으로 재방문 유도 | 지식의 재활용과 선순환 |

### 1-2) 사용자 여정

정보 발견부터 저장, 관리, 재방문까지의 흐름을 설계했습니다.

- **발견** : 웹 서핑 중 유용한 정보를 포착하는 시작점
- **수집 :** 확장 프로그램을 통한 1초 저장 인터랙션
- **분류** : 대시보드에서 태그와 메모를 통한 지식 체계화
- **재회** : 리마인더를 통한 잊힌 정보의 재발견

## 2. 디자인 시스템

사용자의 가독성을 최우선으로 고려하며, 서비스의 신뢰도를 높이기 위한 시스템을 구축했습니다.

### 2-1) 컬러 가이드라인

- Primary (`#345282`)
    
    서비스의 정체성을 담은 Deep Blue 컬러를 메인으로 채택하여 사용자에게 신뢰감과 전문적인 인상을 전달합니다.
    
- Neutral
    
    정보의 깊이를 조절하기 위해 Background (`#F4F7FA`)와 Surface (`#FFFFFF`)를 정의했습니다. 텍스트는 중요도에 따라 Main (`#333333`)과 Sub (`#888888`)로 구분하여 시각적 위계를 확보했습니다.
    
- Tag System
    
    장시간 사용 시 발생할 수 있는 시각적 피로도를 낮추기 위해 채도가 낮고 명도가 높은 7가지 파스텔 톤의 전용 컬러 팔레트를 구축했습니다.
    
- Action & Info
    
    상태 변화를 직관적으로 인지할 수 있도록 긴급 정보는 Warning (`#FF6B6B`), 안내 정보는 Info (`#2194F3`) 컬러를 사용하여 명확한 피드백을 제공합니다.
    

### 2-2) 타이포그래피 및 컴포넌트

다양한 디지털 환경(Web, Extension)에서 가독성이 검증된 INTER을 메인 서체로 사용했습니다.

- 위계 설계
    
    정보의 중요도에 따라 Headline (35px Bold)부터 Body (11px~14px)까지 크기와 굵기를 체계적으로 계층화했습니다.
    
- 시각적 흐름
    
    사용자가 화면에 진입했을 때, 가장 핵심적인 정보를 먼저 인지하고 상세 내용(Body)으로 시선이 자연스럽게 흐르도록 유도하여 인지 부하를 최소화했습니다.
    
- 컴포넌트 자산화
    
    버튼, 입력 폼, 카드 UI 등을 피그마의 Component 및 Variant 기능을 활용해 시스템화했습니다. 이는 디자인 일관성을 유지할 뿐만 아니라, 프론트엔드 개발자가 공통 컴포넌트 단위로 코드를 설계하는 데 기여했습니다.

## 3. 핵심 화면 설계 및 UX 포인트

### 1) 전반적인 디자인 시스템 및 분위기

- White & Light Gray 베이스로 정보 가독성 극대화
- 라운딩 카드 UI와 파스텔 톤을 사용하여 딱딱한 자료실이 아닌 친근한 서재 분위기 조성

### 3-2) 대시보드 : 직관적인 현황 브리핑

- 오늘 저장한 글, 미열람 수 등을 상단 위젯에 배치해 접속 즉시 상태 파악 가능
- 정보 성격에 따른 카드 모듈화로 시각적 혼선 방지

### 3-3) 태그 관리 : 인지 피로도 감소와 행동 유도

- 낮은 채도의 파스텔 톤으로 장시간 사용 시 눈의 피로 감소
- 태그를 '조작 가능한 입체적 객체'로 디자인하여 능동적인 분류 행동 유도

### 3-4) 상세 화면 : 효율적인 정보 가공

- Split View: 원문 열람과 동시에 메모/요약을 참고할 수 있는 화면 분할 레이아웃
- 원문의 맥락을 유지하며 정보를 가공할 수 있는 멀티태스킹 최적화

### 3-5) 리마인더 : 심리적 압박을 줄이는 가이드

- 흐름 중심의 시각화로 업무 우선순위를 자연스럽게 파악하도록 유도
- 긴급 정보는 상단 진한 파란색 배너로 처리하여 정보 누락 방지 및 시각적 위계 확립

### 3-6) 설정 : 게슈탈트 원리를 활용한 구조화

- 근접성의 원리: 연관된 기능끼리 박스 형태로 묶어 인지 복잡도 감소
- 복잡한 경로를 단순화하여 원하는 기능을 즉각적으로 찾을 수 있게 설계

## 5. 리스크 관리 및 해결

디자인 과정에서 발생할 수 있는 사용성 저해 요소를 4가지 관점으로 분석하고, 이를 해결하기 위한 디자인적 대응 방안을 구축했습니다.

[Quick Summary: 리스크 대응 요약]

| 리스크 항목 | 발생한 문제 | 디자인적 대응 및 해결 |
| --- | --- | --- |
| 사용성 | 카드 내 조작 버튼이 많아 발생하는 시각적 소음 | 호버 액션 : 중요도 낮은 기능은 마우스 오버 시 노출 |
| 심미성 | 무분별한 색상 사용으로 인한 톤앤매너 붕괴 | 파스텔 프리셋 : 심미성이 검증된 7종 컬러 강제 적용 |
| 효율성 | 저장 절차의 번거로움으로 인한 수집 이탈 | Zero-effort UX : 자동 수집 및 추가 입력 선택화 |
| 신뢰성 | AI 요약본과 원문 사이의 팩트 체크 어려움 | 원문 바로가기 : 리마인더 카드 내 직접 버튼 배치 |

### 5-1) 사용성 리스크

사용자가 기능을 직관적으로 이해하지 못하거나 조작에 실패하여 서비스에서 이탈할 위험을 관리했습니다.

- 시각적 소음으로 인한 기능 혼선
    
     카드 내 수정, 삭제 등 조작 버튼이 많아 화면이 지저분해 보일 수 있는 위험이 있습니다. 이를 해결하기 위해 아이콘을 최소화하고 호버 액션을 도입하여, 중요도가 낮은 기능은 사용자가 필요로 하는 순간에만 노출되도록 인터랙션을 보완했습니다.
    
- 조작 실수로 인한 데이터 손실
    
    삭제 버튼 오클릭으로 중요한 데이터가 유실될 위험을 방지하고자 합니다. 삭제 액션 시 반드시 경고 모달을 띄우도록 설계하였으며, 버튼에 Warning Red (`#FF6B6B`) 컬러를 적용하여 시각적으로 주의를 환기했습니다.
    

### 5-2) 시각적/심미적 리스크

디자인 일관성이 깨지거나 정보 밀도로 인해 사용자가 시각적 피로를 느낄 위험을 최소화했습니다.

- 정보 과부하 및 시선 분산
    
    대시보드에 과도한 정보가 노출되어 위계가 무너지는 것을 방지하기 위해 **KPI 중심 카드 UI**를 도입했습니다. 핵심 지표 4가지를 상단에 크게 배치하여 시각적 질서를 정리했습니다.
    
- 톤앤매너 붕괴 및 시력 저하
    
    사용자가 임의로 지정하는 색상으로 인해 심미성이 훼손되거나 눈의 피로가 급증할 위험이 있습니다. 이를 위해 검증된 7종의 파스텔 프리셋을 제공하여 일관성을 유지하였으며, 저조도 환경을 고려한 다크 모드 전용 시스템을 통해 시각적 편안함을 확보했습니다.
    

### 5-3) 기술적 제약 및 효율 리스크

디자인과 실제 구현 기능 사이의 간극으로 인해 사용자의 작업 속도가 저하될 위험을 개선했습니다.

- 저장 및 탐색의 번거로움
    
    입력 폼이 많아 저장을 포기하거나 자료가 방대해져 탐색 효율이 떨어질 위험이 있습니다. 제목/URL 자동 수집 기능을 통해 Zero-effort UX를 구현하였고, 검색창 하단에 필터 칩을 배치하여 한 번의 터치로 분류별 조회가 가능하도록 설계했습니다.
    

### 5-4) 정보 신뢰성 및 인지 리스크

사용자가 정보의 우선순위를 놓치거나 제공되는 내용에 대해 불신을 가질 위험을 설계로 보완했습니다.

- 인지 부하 및 정보 간과
    
    단순 리스트 형태는 마감 기한 파악이 어렵고 중요 정보를 놓칠 위험이 큽니다. 수직 타임라인 UI를 도입해 시간의 선후 관계를 시각화하였고, 최상단에 강렬한 Blue 배너를 고정 배치하여 긴급 정보를 강제로 환기하도록 설계했습니다.
    
- 원본 정보 접근성 저하
    
    알림 목록에서 상세 주소 확인이 어려워 다시 검색해야 하는 번거로움을 해결하기 위해, 리마인더 카드 내에 '원문 바로가기' 버튼을 명시적으로 배치하여 사용자 동선을 최적화했습니다.
    

## 6. 라이선스 및 협업 로그

### 6-1) 사용 라이선스

프로젝트의 시각적 일관성과 법적 안전성을 위해 검증된 오픈 소스 리소스를 활용했습니다.

- Font : Inter / Pretendard (SIL Open Font License)
    
    시스템의 기본 서체인 Inter와 가독성이 높은 Pretendard를 혼용하여 디지털 환경에 최적화된 가독성을 구현했습니다.
    
- Icons : Stratis UI Icons (Free Community License)
    
    일관된 디자인 시스템 유지를 위해 1,000여 종 이상의 무료 아이콘 세트를 기반으로 설계하였으나, 실제 구현 시에는 개발 효율성을 위해 프론트엔드 환경에 최적화된 유사 아이콘 라이브러리를 혼용하여 적용했습니다.
    

### 6-2) 협업 및 의사결정 로그

각 파트와의 협업을 통해 프로젝트의 완성도와 구현 가능성을 동시에 확보했습니다.

| **협업 안건** | **상호 협의 및 발전 과정** | **최종 결과** |
| --- | --- | --- |
| 태그 입력 로직 최적화 | [상황] 확장 프로그램에서 태그를 일일이 선택하거나 관리하는 방식이 구현 리소스를 과다 점유함을 인지했습니다. 
[협업] 개발 파트와 논의하여 사용자가 텍스트를 입력하면 시스템이 기존 DB 내 태그와 비교하여 자동 판별하는 로직을 고안했습니다. 동일한 태그가 있으면 해당 속성(색상, 이름)을 그대로 따르고, 없으면 랜덤 색상으로 새 태그를 자동 생성하도록 설계했습니다. | 복잡한 선택 인터랙션을 배제하고 저장 속도를 비약적으로 높인 '1초 태그 수집' UX를 구현했습니다. |
| 목록 탐색 방식 결정 | [상황] 초기 기획 단계에서 트렌디한 '무한 스크롤'과 전통적인 '페이지네이션' 방식의 도입을 두고 논의했습니다.
