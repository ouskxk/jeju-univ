# 🎨 MySave Design System

> **"웹 서핑의 맥락을 방해하지 않는 수집과, 꺼내보고 싶은 나만의 지식 서재"**
> MySave는 기존 북마크의 정적인 리스트 방식을 탈피하여, 사용자의 사고 체계에 맞춘 유연한 정보 관리 환경을 설계합니다.

---

## 1. 디자인 전략 (Design Strategy)

### 1-1) UX 목표
사용자가 정보를 수집하고 관리하는 과정에서 겪는 실제적인 불편함을 디자인적 사고로 해결하는 데 집중했습니다.

| 사용자 문제 | 해결 방안 | 핵심 가치 |
| :--- | :--- | :--- |
| **복잡한 북마크 과정** | Chrome Extension 도입 | 사용 편리성 |
| **저장 시 흐름 단절** | 단 한 번의 클릭으로 자동 저장 | 방해 없는 수집 환경 |
| **지루한 텍스트 리스트** | 이미지/태그 중심의 카드형 UI | 시각적 명확성 |
| **정보 탐색의 어려움** | 시각적 위계 중심 배치 | 탐색 인지 부하 감소 |
| **잊혀지는 정보들** | 리마인더 & 태그 시스템 | 지속 가능성 |
| **지식의 사장화** | 타임라인 기반 재방문 유도 | 지식의 재활용과 선순환 |

### 1-2) 사용자 여정 (User Flow)
정보 발견부터 저장, 관리, 재방문까지의 유기적인 흐름을 설계했습니다.

* **발견(Discovery)**: 웹 서핑 중 유용한 정보를 포착하는 시작점
* **수집(Collection)**: 확장 프로그램을 통한 1초 저장 인터랙션
* **분류(Curation)**: 대시보드에서 태그와 메모를 통한 지식 체계화
* **재회(Revisit)**: 리마인더를 통한 잊힌 정보의 재발견

## 2. 디자인 시스템 (UI Foundation)

사용자의 가독성을 최우선으로 고려하며, 서비스의 신뢰도를 높이기 위한 시스템을 구축했습니다.

### 2-1) 컬러 가이드라인
* **Primary (`#345282`)**: 서비스의 정체성을 담은 Deep Blue를 채택하여 신뢰감과 전문성을 전달합니다.
* **Neutral**: 배경(` #F4F7FA `)과 표면(` #FFFFFF `)을 정의하고, 텍스트는 위계에 따라 Main(` #333333 `)과 Sub(` #888888 `)로 구분했습니다.
* **Tag System**: 시각적 피로도를 낮추기 위해 채도가 낮고 명도가 높은 7가지 파스텔 톤 팔레트를 구축했습니다.
* **Action & Info**: 긴급 정보는 Warning(` #FF6B6B `), 안내 정보는 Info(` #2194F3 `)를 사용하여 명확한 피드백을 제공합니다.

### 2-2) 타이포그래피 및 컴포넌트
* **Main Typeface**: 디지털 환경에서 가독성이 검증된 **INTER** 서체를 사용했습니다.
* **위계 설계**:Headline (35px Bold)부터 Body (11px~14px)까지 체계적으로 계층화했습니다.
* **시각적 흐름**: 핵심 정보를 먼저 인지하고 상세 내용으로 시선이 흐르도록 유도하여 인지 부하를 최소화했습니다.
* **컴포넌트 자산화**: Figma의 **Component 및 Variant** 기능을 활용해 버튼, 카드 UI 등을 시스템화하여 디자인 일관성과 개발 생산성을 높였습니다.

## 3. 핵심 화면 설계 및 UX 포인트

### 3-1) 전반적인 분위기
* White & Light Gray 베이스를 사용하여 정보의 가독성을 극대화했습니다.
* 라운딩 카드 UI와 파스텔 톤을 매치하여 '꺼내보고 싶은 나만의 서재' 분위기를 조성했습니다.

### 3-2) 대시보드 : 직관적인 현황 브리핑
* 오늘 저장한 글, 미열람 수 등을 상단 위젯에 배치해 접속 즉시 상태 파악이 가능합니다.
* 정보 성격에 따른 카드 모듈화로 시각적 혼선을 방지했습니다.

### 3-3) 태그 관리 : 인지 피로도 감소와 행동 유도
* 낮은 채도의 파스텔 톤을 사용하여 장시간 사용 시의 눈 피로도를 낮췄습니다.
* 태그를 '조작 가능한 입체적 객체'로 디자인하여 사용자의 능동적인 분류를 유도합니다.

### 3-4) 상세 화면 : 효율적인 정보 가공
* **Split View**: 원문 열람과 동시에 메모/요약을 참고할 수 있는 화면 분할 레이아웃을 적용했습니다.
* 원문의 맥락을 유지하며 정보를 가공할 수 있는 멀티태스킹 최적화 환경입니다.

### 3-5) 리마인더 : 심리적 압박을 줄이는 가이드
* 흐름 중심의 시각화를 통해 업무 우선순위를 자연스럽게 파악하도록 유도했습니다.
* 긴급 정보는 상단 진한 파란색 배너로 처리하여 정보 누락을 방지하고 위계를 확립했습니다.

### 3-6) 설정 : 게슈탈트 원리를 활용한 구조화
* **근접성의 원리**: 연관된 기능끼리 박스 형태로 묶어 인지 복잡도를 낮췄습니다.
* 원하는 기능을 즉각적으로 찾을 수 있도록 복잡한 경로를 단순화했습니다.

## 5. 리스크 관리 및 해결 (Risk Management)

디자인 과정에서 발생할 수 있는 사용성 저해 요소를 분석하고, 이를 해결하기 위한 대응 방안을 구축했습니다.

| 리스크 항목 | 발생한 문제 (Pain Point) | 디자인적 대응 및 해결 (Solution) |
| :--- | :--- | :--- |
| **사용성** | 카드 내 과도한 조작 버튼으로 인한 시각적 소음 | **호버 액션**: 중요도 낮은 기능은 마우스 오버 시에만 노출 |
| **심미성** | 무분별한 색상 사용으로 인한 톤앤매너 붕괴 | **파스텔 프리셋**: 심미성이 검증된 7종 컬러 강제 적용 |
| **효율성** | 번거로운 저장 절차로 인한 사용자의 수집 이탈 | **Zero-effort UX**: 자동 수집 기능을 통한 입력 최소화 |
| **신뢰성** | AI 요약본과 원문 사이의 팩트 체크 어려움 | **원문 바로가기**: 리마인더 카드 내 직접 버튼 배치 |

### 5-1) 상세 리스크 분석
* **사용성 리스크**: 아이콘 최소화 및 호버 액션 도입을 통해 기능 혼선을 방지하고, 삭제 버튼에 경고 모달 및 Warning Red 컬러를 적용하여 데이터 유실을 방지했습니다.
* **시각적/심미적 리스크**: KPI 중심 카드 UI로 시각적 위계를 정리하고, 다크 모드 전용 컬러 시스템을 설계하여 장시간 사용 시의 시력을 보호했습니다.
* **기술적/효율 리스크**: 제목/URL 자동 수집과 검색창 하단 필터 칩 배치를 통해 탐색 및 수집 효율성을 극대화했습니다.
* **정보 신뢰성/인지 리스크**: 수직 타임라인 UI로 인지 부하를 낮추고, 타임라인 최상단에 강렬한 Blue 배너를 고정하여 긴급 정보를 환기했습니다.

## 7. 프로젝트 후기 (Retrospective)

본 프로젝트를 통해 Figma의 핵심 기능인 **컴포넌트, 베리언츠, 오토 레이아웃**을 실무 수준으로 익히고 적용해 볼 수 있었습니다. 반복되는 UI 요소들을 시스템화하며 디자인 일관성의 중요성을 깊이 체감했습니다.

정교한 인터랙티브 프로토타입을 설계하며 사용자의 동선을 직접 검증해 본 과정은 매우 보람찼습니다. 다만, 개발 구현 환경에 맞춰 디자인을 수정하는 과정에서 대응 속도가 지연되었던 점은 아쉬움으로 남습니다. 이를 통해 **'유지보수가 용이한 디자인 시스템'**의 필요성을 절실히 느꼈으며, 앞으로의 협업에서 더 긴밀한 소통과 견고한 가이드를 구축해 나갈 소중한 자산을 얻었습니다.



## 5. 리스크 관리 및 해결 (Risk Management)

디자인 과정에서 발생할 수 있는 사용성 저해 요소를 분석하고, 이를 해결하기 위한 디자인적 대응 방안을 구축했습니다.

### [Quick Summary: 리스크 대응 요약]

| 리스크 항목 | 발생한 문제 (Pain Point) | 디자인적 대응 및 해결 (Solution) |
| :--- | :--- | :--- |
| **사용성** | 카드 내 과도한 조작 버튼으로 인한 시각적 소음 발생 | **호버 액션**: 중요도 낮은 기능은 마우스 오버 시에만 노출 |
| **심미성** | 무분별한 색상 사용에 따른 서비스 톤앤매너 붕괴 위험 | **파스텔 프리셋**: 심미성이 검증된 전용 7종 컬러 강제 적용 |
| **효율성** | 번거로운 저장 절차로 인한 사용자의 수집 이탈 발생 | **Zero-effort UX**: 자동 수집 기능을 통한 입력 단계 최소화 |
| **신뢰성** | 리마인더 목록 내 상세 주소 확인 및 팩트 체크의 어려움 | **원문 바로가기**: 리마인더 카드 내 직접 버튼 배치로 동선 최적화 |

---

### 사용성 리스크
사용자가 기능을 직관적으로 이해하지 못하거나 조작에 실패하여 서비스에서 이탈할 위험을 관리했습니다.

* **시각적 소음으로 인한 기능 혼선**
    * 카드 내 수정, 삭제 등 조작 버튼이 많아 화면이 지저분해 보일 수 있는 위험이 있습니다. 이를 해결하기 위해 아이콘을 최소화하고 **호버 액션(Hover Action)**을 도입하여, 중요도가 낮은 기능은 사용자가 필요로 하는 순간에만 노출되도록 인터랙션을 보완했습니다.
* **조작 실수로 인한 데이터 손실**
    * 삭제 버튼 오클릭으로 중요한 데이터가 유실될 위험을 방지하고자 합니다. 삭제 액션 시 반드시 **경고 모달**을 띄우도록 설계하였으며, 버튼에 **Warning Red (`#FF6B6B`)** 컬러를 적용하여 시각적으로 주의를 환기했습니다.

### 시각적/심미적 리스크
디자인 일관성이 깨지거나 정보 밀도로 인해 사용자가 시각적 피로를 느낄 위험을 최소화했습니다.

* **정보 과부하 및 시선 분산**
    * 대시보드에 과도한 정보가 노출되어 위계가 무너지는 것을 방지하기 위해 **KPI 중심 카드 UI**를 도입했습니다. 핵심 지표 4가지를 상단에 크게 배치하여 시각적 질서를 정리했습니다.
* **톤앤매너 붕괴 및 시력 저하**
    * 사용자가 임의로 지정하는 색상으로 인해 심미성이 훼손되거나 눈의 피로가 급증할 위험이 있습니다. 이를 위해 검증된 **7종의 파스텔 프리셋**을 제공하여 일관성을 유지하였으며, 저조도 환경을 고려한 **다크 모드 전용 시스템**을 통해 시각적 편안함을 확보했습니다.

### 기술적 제약 및 효율 리스크
디자인과 실제 구현 기능 사이의 간극으로 인해 사용자의 작업 속도가 저하될 위험을 개선했습니다.

* **저장 및 탐색의 번거로움**
    * 입력 폼이 많아 저장을 포기하거나 자료가 방대해져 탐색 효율이 떨어질 위험이 있습니다. 제목/URL 자동 수집 기능을 통해 **Zero-effort UX**를 구현하였고, 검색창 하단에 **필터 칩(Filter Chips)**을 배치하여 한 번의 터치로 분류별 조회가 가능하도록 설계했습니다.

### 정보 신뢰성 및 인지 리스크
사용자가 정보의 우선순위를 놓치거나 제공되는 내용에 대해 불신을 가질 위험을 설계로 보완했습니다.

* **인지 부하 및 정보 간과**
    * 단순 리스트 형태는 마감 기한 파악이 어렵고 중요 정보를 놓칠 위험이 큽니다. **수직 타임라인 UI**를 도입해 시간의 선후 관계를 시각화하였고, 최상단에 **강렬한 Blue 배너**를 고정 배치하여 긴급 정보를 강제로 환기하도록 설계했습니다.
* **원본 정보 접근성 저하**
    * 알림 목록에서 상세 주소 확인이 어려워 다시 검색해야 하는 번거로움을 해결하기 위해, 리마인더 카드 내에 **'원문 바로가기' 버튼**을 명시적으로 배치하여 사용자 동선을 최적화했습니다.
